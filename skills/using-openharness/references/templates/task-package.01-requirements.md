# Requirements

> 章节标题保留英文；正文默认使用中文；命令、状态值、YAML 键名、文件名与路径保持英文。

## Goal
用中文写清楚这项工作的目标，说明希望达到的结果，并写明单一成功指标。

## Problem Statement
用中文说明当前痛点、缺口、冲突或风险，解释为什么现在要做这件事；至少写出目标用户、核心场景和为什么现在要做。

## Required Outcomes
1. 用中文写出必须交付的结果，按可验证的粒度列清楚，并为核心结果写最小 `acceptance criteria`。

## Non-Goals
- 用中文写明本轮明确不做什么，避免范围漂移；至少写一个看起来相关但不属于本轮的 `counterexample`。

## Constraints
- 用中文写明协议限制、兼容性约束、依赖条件、`cost cap` 和不可变边界。
